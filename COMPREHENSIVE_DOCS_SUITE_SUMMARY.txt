📋 DOCSTRING STANDARDS
================================================================================

All docstrings use Google-style format for Sphinx compatibility.

MODULE DOCSTRING:
  """Module name: Brief description.
  
  Longer description of what module does.
  
  Key Components:
    - Component 1: Description
    - Component 2: Description
  
  Architecture:
    [Brief architecture description]
  
  Example:
    >>> from module import function
    >>> result = function(param)
  """

FUNCTION DOCSTRING:
  """Brief one-line description.
  
  Longer description explaining what function does.
  
  Args:
    param1 (str): Description of param1
    param2 (int): Description of param2
  
  Returns:
    Dict[str, Any]: Description of return value
  
  Raises:
    ValueError: When param1 is invalid
    TypeError: When param2 is not int
  
  Example:
    >>> result = function("test", 42)
    >>> print(result)
    {'success': True, 'data': {...}}
  """

CLASS DOCSTRING:
  """Brief one-line description of class.
  
  Longer description of class purpose.
  
  Attributes:
    attr1 (str): Description of attr1
    attr2 (int): Description of attr2
  
  Example:
    >>> obj = MyClass("test", 42)
    >>> obj.method()
    {'result': 'value'}
  """

TOOL FUNCTION DOCSTRING:
  """Perform CRUD operations on entities.
  
  This is an MCP tool that agents use to...
  
  Tool Operations:
    - 'create': Create new entity
    - 'read': Read entity by ID
    - 'update': Update entity
    - 'delete': Delete entity
  
  Args:
    operation (str): Operation to perform
    entity_type (str): Type of entity
    entity_id (Optional[str]): Entity ID
    properties (Optional[Dict]): Entity properties
  
  Returns:
    Dict[str, Any]: Tool response
  
  Agent Reasoning:
    Agents use this tool to:
    1. Create entities
    2. Read entities
    3. Update entities
    4. Delete entities
  
  Example:
    >>> result = await entity_operation(
    ...     operation='create',
    ...     entity_type='document',
    ...     properties={'title': 'My Doc'}
    ... )
  """
