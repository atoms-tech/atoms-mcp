Successfully migrated from Standalone Connect pattern to native FastMCP AuthKit pattern.

## What Changed

### Before (Standalone Connect)

**Complexity**: HIGH
- Custom `PersistentAuthKitProvider` class
- Custom OAuth endpoint handlers
- Custom session middleware
- Manual base URL detection
- ~500 lines of custom auth code across 3 files

```python
# server.py (old)
from auth.persistent_authkit_provider import PersistentAuthKitProvider

auth_provider = PersistentAuthKitProvider(
    authkit_domain=authkit_domain,
    base_url=base_url,
    required_scopes=None,
)

# Custom OAuth endpoints
@mcp.custom_route("/.well-known/oauth-protected-resource")
async def _oauth_protected_resource_handler(request):
    # ... 20 lines of custom logic

@mcp.custom_route("/.well-known/oauth-authorization-server")
async def _oauth_authorization_server_handler(request):
    # ... 15 lines of custom logic

# Custom session middleware
from auth.session_middleware import SessionMiddleware
```

### After (Native AuthKit)

**Complexity**: LOW
- Native FastMCP `AuthKitProvider`
- Auto-generated OAuth endpoints
- Built-in session management
- Auto-detected base URL
- ~5 lines of config

```python
# server.py (new)
from fastmcp.auth import AuthKitProvider

auth_provider = AuthKitProvider(
    authkit_domain=authkit_domain,
    # FastMCP handles everything else automatically
)

# OAuth endpoints auto-generated by FastMCP:
# - /.well-known/oauth-protected-resource
# - /.well-known/oauth-authorization-server
# - /auth/start
# - /auth/complete
```
