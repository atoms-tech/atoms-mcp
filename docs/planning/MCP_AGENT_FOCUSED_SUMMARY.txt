🎯 CORE INSIGHT
================================================================================

MCP IS NOT AN API - IT'S A UI FOR AGENTS

Traditional API documentation focuses on:
  ❌ Parameters and types
  ❌ Function signatures
  ❌ Return values
  ❌ Error codes

Agent-focused MCP documentation focuses on:
  ✅ Agent workflows
  ✅ Agent reasoning
  ✅ Multi-turn conversations
  ✅ Real-world demonstrations
  ✅ Terminal recordings
  ✅ Error recovery

🎬 DEMONSTRATION TYPES
================================================================================

TYPE 1: VHS TERMINAL RECORDINGS
  What: Record terminal interactions as GIFs
  Tool: vhs (charmbracelet)
  Output: Shareable GIFs, embeddable in docs
  Duration: ~30 seconds each
  Count: 17 total

TYPE 2: CODE EXAMPLES WITH REPL OUTPUT
  What: Show code + actual output
  Format: Markdown with code blocks
  Count: 50+ examples
  
  Example:
    >>> from atoms_mcp import AtomsMCP
    >>> client = AtomsMCP()
    >>> result = await client.call_tool(...)
    >>> print(result)
    {"success": true, "data": {...}}

TYPE 3: MULTI-TURN CONVERSATIONS
  What: Show agent reasoning and multi-step workflows
  Format: Conversation transcript
  Count: 17 examples
  
  Example:
    User: "Create 3 documents"
    Agent: [creates 3 entities]
    Agent: "Created 3 documents"
    
    User: "Tag them as 'draft'"
    Agent: [updates all 3]
    Agent: "Tagged all as 'draft'"

TYPE 4: AGENT REASONING TRACES
  What: Show how agents think
  Format: Structured reasoning steps
  Count: 17 examples
  
  Example:
    Agent: "User wants to create entities"
    Agent: "I'll use entity_operation tool"
    Agent: "Parameters: operation='create'"
    Agent: [calls tool]
    Agent: "Success\! Created entity ent_123"

TYPE 5: CURSOR/CLAUDE AGENT RECORDINGS
  What: Live agent interaction with Atoms MCP
  Tool: asciinema, ffmpeg, obs
  Output: Playable videos, shareable links
  Count: 5-10 recordings

================================================================================
🛠️ TOOLS FOR CREATING DEMONSTRATIONS
================================================================================

VHS (Terminal Recording)
  Install: brew install charmbracelet/tap/vhs
  Usage: vhs < demo.tape
  Output: demo.gif
  
  Example:
    Output: create_entity.gif
    Set FontSize 16
    Set Width 1200
    Set Height 600
    
    Type "python3"
    Enter
    Sleep 500ms
    
    Type "from atoms_mcp import AtomsMCP"
    Enter
    Sleep 500ms
    
    Type "client = AtomsMCP()"
    Enter
    Sleep 500ms
    
    Type "result = await client.call_tool(...)"
    Enter
    Sleep 1s
    
    Type "print(result)"
    Enter

ASCIINEMA (Terminal Recording)
  Install: brew install asciinema
  Record: asciinema rec demo.cast
  Play: asciinema play demo.cast
  Share: asciinema upload demo.cast

CURSOR AGENT RECORDING
  Setup: Connect Cursor to Atoms MCP
  Enable: Agent mode
  Record: Terminal output
  Tools: asciinema, ffmpeg, obs

CLAUDE AGENT RECORDING
  Setup: Connect Claude to Atoms MCP
  Record: Conversation + tool calls
  Tools: asciinema, ffmpeg, obs

🎯 KEY DIFFERENCES FROM TRADITIONAL API DOCS
================================================================================

ASPECT                  TRADITIONAL API         AGENT-FOCUSED MCP
────────────────────────────────────────────────────────────────────────────
Focus                   Parameters, types       Agent workflows, reasoning
Examples                Code snippets           Terminal recordings
Demonstrations          None                    vhs, asciinema, recordings
Reasoning               Not shown               Agent thinking traces
Interaction             Single calls            Multi-turn conversations
Use Cases               Generic                 Real-world agent scenarios
Learning Style          Read docs               Watch demonstrations
Engagement              Low                     High
Maintenance             High (docs drift)       Low (recordings are truth)
